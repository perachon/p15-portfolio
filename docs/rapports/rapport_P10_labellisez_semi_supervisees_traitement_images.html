Rapport de conduite de projet AI Engineering — Semi-supervisé en traitement d'images / BrainScanAI

1. Contexte et objectif

  • Organisation (scénario) : CurelyticsIA, startup e-santé.
  • Cas d’usage R&D : détection tumeur vs normal sur des images IRM (classification binaire).
  • Contrainte clé : peu de labels (annotation coûteuse) + grand volume d’images non labellisées.
  • Enjeu méthodologique : éviter la fuite de données (test strict, décisions prises sur train/val).

Note : les sources présentent cette approche comme une démonstration technique (pas un diagnostic médical complet).

2. Données (extraits)

  • Total : 1506 images.
  • Labellisées : 100 (50 normal / 50 cancer).
  • Non labellisées : 1406.
  • Split : train/val/test sur le labellisé, avec test strict réservé à l’évaluation finale.

3. Pipeline (vue d’ensemble)

3.1 Contrôle qualité (QC)

  • Statistiques par image : taille + intensité (moyenne, dispersion, percentiles).
  • Outliers et quasi-doublons : analyse via la similarité dans l’espace des embeddings (cosinus).
  • Garde-fou : ne jamais supprimer automatiquement les images labellisées.

3.2 Prétraitement & embeddings

  • Prétraitement : conversion en RGB, redimensionnement 224×224, normalisation ImageNet.
  • Extracteur : ResNet50 pré-entraîné (extracteur figé).
  • Sortie : 2048 features par image (matrice 1506×2048).

3.3 Clustering & évaluation sans fuite

  • Méthodes testées : KMeans (principal) + DBSCAN (comparaison).
  • Évaluation (train/val labellisé uniquement) : ARI (Adjusted Rand Index) entre clusters et classes.
  • Le test n’est jamais utilisé pour guider les choix (anti fuite de données).

3.4 Pseudo-labels & semi-supervisé

  • Mapping cluster → classe appris uniquement sur le train labellisé (sans toucher au test).
  • Pseudo-labels générés sur le non labellisé, puis filtrage de confiance pour réduire le bruit.
  • Comparaison : baseline supervisée (labeled only) vs semi-supervisé (pré-train pseudo-labels → fine-tune labels).
  • Robustesse : exécutions multi-seeds (moyenne / écart-type).

4. Résultats (extraits)

  • Métriques : F1 macro + matrice de confusion (accuracy en complément).
  • Pseudo-labels : 1406 images pseudo-labellisées puis filtrage (≈ un quart conservé, autour de 350 images).
  • Test final (extraits) : supervisé ~95% accuracy / ~0.95 F1 macro ; semi-supervisé ~90% accuracy / ~0.90 F1 macro.
  • Message clé présenté : performances comparables dans l’exercice et stabilité accrue (variance plus faible) en semi-supervisé filtré.

5. Budget & passage à l’échelle (extraits)

  • Budget mentionné pour la labellisation IA du dataset : 300 €.
  • Projection : 5 000 € pour labelliser 4 millions d’images.
  • Recommandations : pipeline batch relançable, traçabilité (index, artefacts, versions), monitoring simple (QC + dérive).

6. Points d’attention & suites possibles (extraits)

  • Risques : biais, outliers/doublons, fuite de données.
  • Suites proposées : normalisation spécifique IRM, calibration, validation externe, annotation progressive (active learning).
