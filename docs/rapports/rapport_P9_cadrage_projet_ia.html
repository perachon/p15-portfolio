Rapport de conduite de projet AI Engineering — Cadrage d’un projet IA / Fashion‑Insta

1. Contexte et besoin

  • Entreprise (scénario) : Fashion‑Insta (magasins + e-commerce), marché concurrentiel, enjeu d’expérience client digitale.
  • Problème : difficulté pour les clients à retrouver des articles similaires à un style via des mots-clés/filtres.
  • Opportunité : utiliser la photo comme point d’entrée pour recommander des articles similaires.

2. Objectif du PoC IA

  • Faisabilité : vérifier que l’IA peut comprendre un style à partir d’une photo et proposer des recommandations pertinentes.
  • Valeur métier : valider l’intérêt de l’IA avant d’investir dans un MVP/produit complet.
  • Périmètre limité : PoC (pas un produit final).
  • Fonction centrale : entrée = photo utilisateur ; sortie = top N d’articles visuellement similaires ; analyse purement visuelle (pas de personnalisation dans le PoC).

3. Approche technique (PoC)

  • Modèle vision pré-entraîné (Vision Transformer / CNN).
  • Extraction d’embeddings (empreintes visuelles).
  • Recherche de similarité dans le catalogue (ex. distance cosinus) pour retourner les produits les plus proches.

Approche alternative (mentionnée)

  • Multimodal image + texte (type CLIP / LLM multimodal) : plus riche sémantiquement, mais plus coûteux et plus complexe ; option selon résultats.

4. Données et outils

  • Données : images produits du catalogue ; datasets publics de mode en complément si besoin.
  • Cloud : Microsoft Azure (partenaire cloud).
  • PoC : pas de données personnelles clients (extrait).

5. Critères de succès (Go / No Go)

  • Métier : recommandations jugées pertinentes par les équipes métier.
  • Technique : Recall@K / Precision@K (top 5) ; temps d’inférence < 500 ms.
  • Décision : Go/No Go sur validation croisée des critères techniques et métier.

6. Déroulé, staffing et timeline

  • Déroulé PoC : cadrage & préparation des données → tests de modèles → évaluation métier → synthèse & recommandations.
  • Durée PoC (extrait) : 4 à 8 semaines.
  • Profils (extraits) : 1 Data Scientist ; 1 AI Engineer / Data Engineer ; 1 référent métier mode.
  • Timeline globale (extraits) : trajectoire progressive ~6 mois (cadrage → PoC → MVP → Run/pilote) avec Copil/Coproj.

7. System design (vue d’ensemble)

  • Ingestion : photo envoyée via API sécurisée, stockage temporaire.
  • Inférence : modèle vision → embedding.
  • Recherche vectorielle : comparaison embedding vs catalogue indexé (embeddings pré-calculés) + top N.
  • Règles métier : filtrage avant retour client.
  • Scalabilité : services managés Azure ; point critique = service d’inférence (latence & coût compute).

8. Coûts, ROI (extraits)

  • Coûts RH : ~116 500 €.
  • Coûts techniques one-shot : ~10 000 €.
  • Coûts récurrents Run : ~20 000 € / an.
  • Coût total première année : ~146 500 €.
  • ROI (hypothèses marketing, extrait) : gains annuels ~468 000 € ; rentabilité dès la première année.

9. RGPD : risques et mesures (extraits)

Risques

  • Images pouvant contenir des personnes ; ré-identification indirecte.
  • Réutilisation non maîtrisée ; conservation excessive ; exposition via APIs externes.
  • Données personnelles indirectes (nom/email/identifiant de compte).

Mesures

  • Gouvernance : consentement explicite ; finalité recommandation ; pas d’entraînement de modèles tiers/LLMs sur images utilisateurs.
  • Technique : stockage séparé, suppression automatique, chiffrement au repos/en transit, logs anonymisés.
  • Minimisation : privilégier embeddings ; suppression de l’image brute après traitement.
  • Fournisseurs : pas d’envoi d’images à des APIs publiques ; modèles hébergés sur Azure.
  • DPIA : envisagée avant déploiement à grande échelle (phase MVP).
