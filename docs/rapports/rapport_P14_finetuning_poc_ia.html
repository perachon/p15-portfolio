Rapport de conduite de projet AI Engineering — P14 (Fine-tuning LLM / POC triage médical — CHSA)

1. Contexte et objectif

  • Client (scénario) : Centre Hospitalier Saint‑Aurélien (CHSA), urgences surchargées.
  • But : POC d’un agent IA de triage médical (priorité urgence maximale / modérée / différée).
  • Attendus : questionnaire adaptatif, explications, intégration SIH, traçabilité pour audit.
  • Échéance : 4 semaines.

2. Stratégie en phases (extraits)

  • Phase 1 : validation conceptuelle avec Qwen3‑1.7B‑Base.
  • Phase 2 : spécialisation via SFT + LoRA puis alignement par préférences (DPO).
  • Phase 3 : projection industrielle (modèles plus grands 32B+ en cas de validation).

3. Données et gouvernance (extraits)

  • Datasets cités : MediQA, FrenchMedMCQA, MedQuAD, UltraMedical‑Preference.
  • Objectif SFT : ~5 000 paires instruction‑réponse.
  • DPO : paires préférentielles (réponses validées/non validées).
  • RGPD : anonymisation + justification ; recommandation d’outil Presidio.
  • Formats : JSONL / Hugging Face Datasets + schéma de métadonnées.

4. Entraînement et traçabilité expérimentale (extraits)

  • Outils : PyTorch, Transformers, PEFT (LoRA) ; tracking via MLflow (ou W&B).
  • Bonnes pratiques : petits runs LoRA, checkpoints, documentation (hyperparams + seed).
  • Contrôles : métriques cliniques + sécurité (hallucinations, recommandations dangereuses).

5. API, sécurité et traçabilité (extraits)

  • Garde‑fous : logique de red flags (priorité forcée + réponse prudente, génératif court‑circuité).
  • Traçabilité : audit des interactions en SQLite + endpoint /audit/{interaction_id}.
  • Ordre de démo (extrait) : GET /health → POST /triage (standard) → POST /triage (red flag) → GET /audit/{interaction_id}.

6. Déploiement, performance et CI/CD (extraits)

  • Déploiement : Docker + FastAPI ; trajectoire vLLM pour inférence optimisée.
  • CI/CD : GitHub Actions (lint Ruff, tests Pytest, build Docker).
  • Latence (extraits) : local réel P50 ~13,5 s ; cloud stub P50 ~0,23 s.

7. Livrables attendus (extraits)

  • Dataset médical bilingue (HF Datasets/JSONL) versionné.
  • Modèle spécialisé (SFT+LoRA) + alignement (DPO).
  • Endpoint de démonstration optimisé (vLLM) + CI/CD GitHub Actions.
  • Rapport technique + recommandations (20 pages max).
