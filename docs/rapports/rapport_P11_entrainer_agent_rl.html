Rapport de conduite de projet AI Engineering — Entraînez votre agent RL / Eagle‑1 / LunarLander‑v3

1. Contexte et objectif

  • Sujet : apprentissage par renforcement (RL) — apprendre par interaction pour maximiser une récompense cumulative.
  • Environnement : LunarLander‑v3 (atterrisseur “Eagle‑1”).
  • Formulation RL (extraits) : observation = 8 valeurs ; action = 4 actions discrètes ; reward favorise un atterrissage contrôlé et pénalise crash / fuel.
  • Métrique : mean_reward sur 100 épisodes.
  • Critère de succès : mean_reward > 200 stable.

2. Approche

  • Étape 1 : exploration + baseline.
  • Étape 2 : tuning progressif avec évaluation sur 100 épisodes.
  • Étape 3 : “mise en usage” via API + GUI + dashboard + vidéo.

3. Choix d’algorithme

  • Démarrage : DQN (référence initiale, action space discret).
  • Choix final : PPO (plus stable et meilleur score final observé).
  • Meilleur run conservé : PPO avec gamma=0.999.

4. Résultats (extraits)

  • Objectif : mean_reward > 200 / 100 épisodes.
  • Résultat annoncé : ~267 ± 21 (sur 100 épisodes).
  • Best model sauvegardé + historique de runs loggé.

5. Livrables et “mise en usage”

Architecture

  • API : charge le modèle, calcule l’action, joue l’épisode, génère la vidéo, loggue.
  • GUI : client qui appelle l’API, affiche vidéo + résumé reward/steps.
  • Dashboard : lit les logs (CSV) et propose des filtres simples.

API

  • POST /predict : observation → action.
  • POST /play : 1 épisode complet + vidéo + log.
  • Swagger : /docs.

Vidéo

  • Contrainte : 20–30s.
  • Réglages : 30 fps + répétition de frames (ex. frame_repeat=3) pour atteindre ~25–30s sur un épisode classique.

6. Limites et suites (extraits)

  • Limite : variabilité selon la seed et les conditions exactes.
  • Suites : multi‑seeds, analyses d’actions, monitoring plus fin, packaging.
