Rapport de conduite de projet AI Engineering — Requêtez des services IA / ModeTrends

1. Contexte et objectif

  • Organisation (scénario) : ModeTrends, agence de marketing digital dans la mode.
  • Projet : « Fashion Trend Intelligence » (analyse automatisée des tendances sur les réseaux sociaux).
  • Périmètre : segmentation vestimentaire — isoler chaque pièce vestimentaire dans une image.

2. Approche

  • Modèle : SegFormer (segmentation sémantique à base de Transformer ; encoder MiT + decoder léger).
  • Deux modes évalués : traitement local via transformers (sans API) et exécution distante via Hugging Face Inference API.

3. Méthode de validation

  • Métrique : IoU (Intersection over Union).
  • Principe : comparer masques prédits vs masques « ground truth » (par classe et global).
  • Résultat (extrait) : IoU global ≈ 74%.

4. Coût (extraits)

  • Local : 0€ (hors coût machine/disque).
  • Hugging Face Inference API : ~1s/image ; plan Basic à 0.10$/minute au-delà des 30 premières minutes ; estimation donnée ≈ 800$ (≈ 140h de calcul).

5. Défis rencontrés (extraits)

  • Alignement masques prédits / réels : attention au nommage (ex. image_0.png ↔ mask_0.png).
  • Pré-traitement images : éviter la déformation (redimensionnement + padding).
  • Formats : gestion dynamique des extensions (.png, .jpg, …) via .env.
  • Évaluation : comparaison systématique aux masques ground truth.

6. Améliorations possibles (extraits)

  • Filtrage par classe (ignorer classes rares/faible confiance).
  • Lisibilité : overlay + contours + légendes dynamiques.

7. Applications potentielles (extraits)

  • Essayage virtuel / segmentation automatique.
  • Recherche visuelle.
  • Personnalisation marketing.
  • Analyse de tendances (agrégation à grande échelle).
