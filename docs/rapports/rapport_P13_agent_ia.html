Rapport de conduite de projet AI Engineering — Agent IA / ouvertures aux échecs — FFE

1. Contexte et objectif

  • Sujet : agent IA conversationnel interagissant avec une base de connaissances et des outils externes.
  • Concepts cités : LangGraph, RAG + base vectorielle (ex. Milvus), intégration d’APIs et MCP.
  • Client (scénario) : Fédération Française des Échecs (FFE).
  • But : POC d’un agent d’apprentissage des ouvertures aux échecs.
  • Délai : 2 semaines.

2. Fonctionnalités attendues (extraits)

  • Meilleurs coups théoriques, contexte via parties historiques.
  • Vidéos explicatives pertinentes via YouTube API.
  • Évaluation de la position via un moteur spécialisé (ex. Stockfish) si la partie sort de la théorie.

3. Stack et intégrations (extraits)

  • Frontend : Angular + échiquier interactif (ngx-chessboard/material-chessboard).
  • Backend/agent : LangGraph + FastAPI.
  • Données : Milvus (vector search) + MongoDB.
  • APIs/outils : position au format FEN, Lichess (ouvertures/parties), Stockfish, YouTube API.

4. Endpoints et étapes proposées (extraits)

  • Setup : dépôt Git, structure backend/ + frontend/, docker-compose.yml (FastAPI, Milvus, MongoDB, Angular).
  • Recommandation : healthcheck FastAPI sur /api/v1/healthcheck.
  • Endpoints : /api/v1/moves/{fen} (Lichess), /api/v1/evaluate/{fen} (Stockfish), /vector-search (Milvus).
  • Bonnes pratiques : validation FEN via python-chess, gestion d’erreurs/timeouts, attention aux limites API Lichess.

5. Livrables attendus (extraits)

  • Système : Lang Graph, FastAPI, Milvus, MongoDB.
  • Code : dépôt Git.
  • Démo : exécution locale via Docker Compose.
